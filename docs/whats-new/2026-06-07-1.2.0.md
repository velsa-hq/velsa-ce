This release rounds out data management, financial reporting, and
day-to-day operations.

### New

- **Data import.** Bring clients, chart-of-accounts, and bookings in from
  a spreadsheet: upload a CSV, map your columns, preview a dry run that
  validates every row before anything is written, then commit. A
  committed import can be reversed within 7 days.
- **Financial statements.** Balance Sheet and Income Statement now join
  the report library alongside AR aging and budget-vs-actual.
- **Scheduled reports.** Any report can be scheduled to run and be emailed
  on a recurring basis - no need to remember to pull it.
- **Read-only / maintenance mode.** Administrators can temporarily freeze
  all interactive changes from System settings - useful during a data
  import or maintenance window. A banner shows everyone the system is
  read-only; sign-in and imports still work.

### Improved

- **Branded error pages.** 403/404/419/429/500/503 pages now match the
  application's look instead of the generic framework defaults.
- **Stronger sign-in protection.** Privileged (administrator) access can
  require multi-factor authentication, and financial screens are gated to
  the roles that should see them.

### Fixed

- Holds that were never released now expire on schedule and promote the
  next party in line automatically.
- Double-booking is now prevented at the database level, not just in the
  space finder.
