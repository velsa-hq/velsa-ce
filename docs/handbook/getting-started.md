This is the operating manual for Velsa - what each screen does, the
common workflows, and the rules behind the data.

:::video getting-started

## Who this is for

- **Operations staff** entering bookings, contracts, work orders, and run-of-show outlines
- **Sales staff** managing the lead pipeline and converting inquiries to bookings
- **Administrators** managing users, venue permissions, and reading the audit log

## What's in the system

Six modules covering the full event lifecycle:

| Module | What it does |
| --- | --- |
| Pipeline | Sales pipeline for leads and prospect activities |
| Bookings | The core: venues, spaces, dates, statuses |
| Contracts | Drafts, signers, send to client via DocuSign |
| Exhibitors | Booth assignments + orders + payments for trade shows |
| Operations | Ops board, work orders, run-of-show outlines |
| Reports | Saved reports + CSV exports |

The left sidebar in the app mirrors this layout.

## How the handbook is organized

The left sidebar of this handbook groups pages by section. Each page has
a table of contents on the right (on wide screens). Click any heading
inside a page to copy a direct link.

Throughout the app, look for a faint question-mark icon next to a page's
title. Clicking it opens the handbook page for whatever you're looking at,
in a new tab - a shortcut to the docs without leaving your work.

## Practice (Demo / Training) instances

If you see an amber **DEMO** or **TRAINING** ribbon, you're on a *practice*
instance. It has every feature of the real system, but it's deliberately
**non-acting**: emails aren't delivered to anyone, e-signature envelopes
aren't really sent, and card payments aren't really charged. Practice
freely - nothing you do here reaches a real client or real money. (It's
the same app as production, just switched to a safe mode.)
